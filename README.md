# Near-Miss by One Step: A Kernel Theory of Degeneracy in Fine-Tuning Causal Proofs

Comprehensive codebase for training and evaluating decoder-only logical reasoning systems on proof-chain generation, repair, and depth-wise performance analysis.

This repository includes:
- Standard supervised finetuning for proof generation.
- Repair-style finetuning using corrupted proof chains.
- DPO finetuning with chosen vs rejected reasoning chains.
- Inference pipelines (standard and self-generate-plus-repair).
- Assumption 5 verification for near-miss gradient proximity.
- Post-hoc proof quality comparison across model outputs.

## Usage

### Training Pipelines

Standard SFT training:

```bash
python src/train/decoder_train.py
```

Repair-task SFT training:

```bash
python src/train/decoder_train_repair.py
```

Multi-GPU DPO example:

```bash
torchrun --nproc_per_node=4 src/train/decoder_train_dpo.py
```

### Inference Pipelines

1) Standard pipeline: Question -> generated proof -> rule-based label extraction.

```bash
python src/inference/decoder_inference_standard.py
```

2) Repair pipeline: Question -> draft proof -> repaired proof -> rule-based label extraction.

```bash
python src/inference/decoder_inference_repair.py
```

Label extraction utility - To extract the labels from the predicted proof chain
```bash
python src/inference/extract_label_from_proof.py
```

### Assumption 5 Verification

Prepare verification data:

```bash
python src/assumption5/prepare_assumption5_data.py \
  --input data/train_data.parquet \
  --output data/assumption5_data.csv
```

Run Assumption 5 analysis:

```bash
python src/assumption5/verify_assumption5.py \
  --dataset_path data/assumption5_data.csv \
  --model_name Qwen/Qwen2.5-1.5B-Instruct \
  --output_dir results/assumption5 \
  --n_samples 100 \
  --k_near_miss 3
```

## Key Outputs

- `evaluation_metrics.json`: Summary evaluation report (overall accuracy, depth-wise accuracy, valid prediction counts, and sample counts).
- `model-preds.json`: Predicted label for each answer/example (typically 0/1 after label extraction).
- `model-proof-chains.json`: The actual predicted proof chain text generated by the model during inferencefor each example.
- `labels-and-depths.json`: Ground-truth labels and depth values used for evaluation.
